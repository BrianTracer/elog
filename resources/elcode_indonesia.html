ELCode Help


ELCode adalah satu set khusus tag untuk memformat entri elog. Hal ini mirip dengan HMTL, tapi lebih sederhana. tag adalah diapit oleh kurung [dan], bukan & lt; dan & gt ;. Beberapa tag mengubah format teks seperti tebal, ukuran dan warna, sementara tag lain memungkinkan embedding URL dan gambar. Tag ELCode adalah mirip dengan BBCode tags, kadang-kadang juga disebut sebagai vB code.

Tags dapat diulang, seperti

[size=5][color=red][b]Hello[/b][/color][/size]

yang akan menghasilkan Hello. Perhatikan bahwa tag harus diulang dengan benar, hal-hal seperti

[b][color=red]Ini salah![/b][/color]

tidak benar, karena [b] tag dibuka pertama dan karena itu harus ditutup lalu, setelah penutupan [/color] tag, seperti di

[b][color=red]Ini benar[/color][/b]

karakter escape


Kadang-kadang seseorang tidak ingin tag ELCode menjadi ditafsirkan, seperti di forum diskusi di mana seseorang ingin menjelaskan beberapa tag misalnya. Untuk menghindari interpretasi tag, satu menempatkan backslash "\" di depan tag, seperti \[b], bukan [b].

jangkar


Jangkar dapat dimasukkan ke dalam entri dengan [jangkar] anchor_name [/anchor] . Hal ini setara dengan jangkar HTML seperti & lt; nama = " anchor_name "></a> . Jangkar tersebut dapat dirujuk dalam entri dengan elog: & lt; id & gt; # anchor_name di mana <id> adalah nomor ID dari entri (jumlah dari kolom kiri di layar daftar).

mengutip


Ada dua cara mengutip, dengan referensi dan tanpa:

  • Melampirkan beberapa teks dengan [quote] Teks Asli [/quote] menghasilkan kutipan tanpa referensi seperti:

    mengutip:
    Teks Asli

  • Melampirkan beberapa teks dengan [quote = "Mr. Bean"] Teks Asli [/ quote] menghasilkan kutipan dengan referensi seperti:

    Mr. Bean wrote:
    Teks Asli

    Perhatikan bahwa itu adalah wajib untuk menyertakan referensi "Mr Bean" dalam tanda kutip.

Tetap teks lebar [CTRL+O]


Untuk menampilkan teks dengan lebar tetap menggunakan font jenis Courier, apit dengan

[code]Hello[/code]

Ini mungkin berguna dalam daftar kode komputer atau meja untuk meningkatkan pembacaan, seperti

Barang              Harga     Ketersediaan
====                =====     ============

apel               0.50     1000
buah anggur          2.50     100
Sun-kering Tomat   12.50     20

lawan

Barang Harga Ketersediaan
==== ===== ============

apel 0.50 1000
buah anggur 2.50 100
Sun-kering Tomat 12.50 20

Perhatikan bahwa semua tag ELCode lainnya dinonaktifkan dalam [code] tags. ini mungkin bisa membantu untuk menampilkan beberapa kode program atau serupa yang berisi karakter yang akan salah diartikan sebagai tag atau smiley. Jika perilaku ini tidak inginkan, seperti jika beberapa kode program harus dijelaskan dengan huruf tebal dll tag [code1] ... [/code1] dapat digunakan yang memungkinkan lainnya ELCode tag di dalamnya.

Daftar [CTRL+L]


Daftar unordered dapat dibuat dengan tags [list][/list] menyukai

[list]
[*]Ocean
[*]Forest
[*]Desert
[/list]

untuk menghasilkan

  • Ocean
  • Forest
  • Desert

Daftar item baru dimulai dengan [*] , yang dapat dimasukkan melalui keyboard shortcutCTRL-N. Dalam daftar memerintahkan seseorang dapat menentukan apa yang digunakan sebelum setiap item. Untuk membuat daftar satu nomor dapat menggunakan [list=1][/list]:

[list=1]
[*]Ocean
[*]Forest
[*]Desert
[/list]

untuk menghasilkan

  1. Ocean
  2. Forest
  3. Desert

untuk daftar abjad [list = a] [/list] menyukai

[list=a]
[*]Ocean
[*]Forest
[*]Desert
[/list]

untuk menghasilkan

  1. Ocean
  2. Forest
  3. Desert

Kemungkinan lain adalah [list = A] untuk huruf kapital dan [list = I] untuk penomoran Romawi.

Tabel


Tabel dapat dibuat dengan tags [table] [/table] menyukai

[table border="1"]
Satu|Dua
|-
Tiga|Empat
[/table]

untuk menghasilkan

SatuDua
TigaEmpat

Parameter setelah [table ...] secara langsung digunakan dalam HTML <table> tag. Untuk meningkatkan padding sel, pada dapat menambahkan misalnya

[table border="1" cellpadding="20"]
Satu|Dua
|-
Tiga|Empat
[/table]

Untuk Menghasilkan
SatuDua
TigaEmpat

Tabel judul tidak didukung, tetapi dapat disimulasikan dengan menanamkan sel Isi dengan [B]...[/B] tag.

membuat link


Hyperlink atau Locators Resouce Uniform (URL) dapat dibuat dengan berbagai cara:

  • Setiap URL sintaksis benar akan otomatis transformd menjadi link, bahkan tanpa terkemuka http:// di depannya. Jadi midas.psi.ch/elog akan diubah menjadi midas.psi.ch/elog dan http://www.cern.ch akan dikonversi menjadi http://www.cern.ch.

  • Untuk menentukan URL secara eksplisit, apit dengan [url]www.cern.ch[/url]

  • Jika link harus mengandung beberapa teks yang berbeda dari URL sebutkan dengan [url = http://www.cern.ch] Teks [/ url] menyukai di

    pergi ke [url=http://www.cern.ch]CERN[/url]

    yang akan menghasilkan:

    pergi ke CERN

  • Hal yang sama juga alamat email. Mereka dapat digunakan seperti di [email] john.doe@domain.org [/ email] atau hanya sebagai john.dow@domain.org yang akan berubah dengan sendirinya menjadijohn.doe@domain.org.

penyisipan gambar [CTRL+M]


Ada dua cara untuk menyisipkan gambar dalam teks:

  • Gambar dapat disisipkan dalam teks dengan [img] URL [/img] tag, di mana URL harus menunjuk ke sebuah availabe gambar di internet. Catatan bahwa ini hanya bekerja untuk gambar yang dapat diakses melalui browser web. Potret pada hard disk lokal Anda tidak dapat diakses dari luar kecuali jika Anda tidak menjalankan server web publik. Anda dapat misalnya menanamkan logo elog dengan:

    [img] http://midas.psi.ch/elogs/elog.png [/img]

    yang menghasilkan

  • Anda dapat meng-upload gambar sebagai lampiran dalam entri elog Anda dan link untuk itu dengan

    [img] elog:/1[/img]

    di mana elog:/1 singkatan lampiran pertama, elog:/2 untuk dll kedua Jika Anda referensi gambar terlampir dengan cara ini, mereka tidak ditampilkan lagi pada akhir entri.
